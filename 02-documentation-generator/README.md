# Article 2: Building a Documentation Generator

Learn to build a multi-tool agent that reads your codebase and generates comprehensive documentation automatically.

## 📖 Read the Article

**[Read on Medium →](https://medium.com/@alirezarezvani/agent-builders-playbook-documentation-generator)**

## 🎯 What You'll Build

An agent that:
- Scans project directories for source files
- Reads and analyzes code structure
- Generates formatted markdown documentation
- Writes README files to disk

## 🧠 Key Concepts

This article teaches:
1. **Multi-tool orchestration** - Chaining tools together
2. **Tool design** - Writing clear tool descriptions
3. **Iteration limits** - Preventing runaway agents
4. **Workflow planning** - Designing tool sequences

## 💡 What You'll Learn

- Design tools that work together
- Guide agents through complex workflows
- Handle file I/O operations safely
- Control agent iteration and token usage

## 🐛 Troubleshooting

**Common issues:**
- Agent reads too many files → Improve file filtering
- Generated docs are incomplete → Add more specific prompts
- Write permission errors → Check file paths and permissions

See implementation READMEs for detailed troubleshooting.
